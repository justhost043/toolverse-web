When to embed images as Base64

Converting images to Base64 eliminates a separate HTTP request for that image, which can improve performance for small icons, logos, or UI elements that are used on every page. It's especially useful in HTML emails (where external image hosting can be blocked), in single-file HTML documents, in CSS for background images, and in JSON APIs where binary data isn't supported.

When not to use Base64

Base64 encoding increases file size by approximately 33%. For large images or photos, this tradeoff is rarely worth it — the size increase outweighs the benefit of saving one HTTP request. Base64 is best suited for images under 10KB. Larger images should be hosted as separate files and referenced by URL.
