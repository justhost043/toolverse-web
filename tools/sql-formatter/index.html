Why format SQL queries?

Minified or auto-generated SQL is often a single long line with inconsistent capitalization and no indentation. Formatted SQL is dramatically easier to read, debug, and review. When sharing queries with teammates or in code reviews, well-formatted SQL communicates intent clearly. Consistent uppercase keywords are a widely-used SQL convention that makes the structural parts of a query visually distinct from table and column names.
