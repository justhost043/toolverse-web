What is a hash?

A hash function takes any input — a word, a file, a database — and produces a fixed-length string of characters called a hash or digest. The same input always produces the same hash, but even a tiny change to the input produces a completely different hash. This property makes hashes useful for verifying data integrity: if you download a file and its hash matches the one published by the source, you can be confident the file hasn't been altered.

SHA-256 vs SHA-1 vs MD5

SHA-256 is the current gold standard — it's part of the SHA-2 family and is used in SSL/TLS certificates, Bitcoin, and most modern security applications. SHA-1 is older and has known vulnerabilities — it should not be used for security-critical purposes, but is still used for non-security tasks like Git commit hashes. MD5 is broken for security use but still widely used for simple checksums, like verifying that a downloaded file isn't corrupted.

Is it safe to hash passwords here?

This tool should not be used to hash passwords for storage. For passwords, you should use a dedicated password hashing algorithm like bcrypt, Argon2, or scrypt — these are intentionally slow to make brute-force attacks computationally expensive. SHA-256 and MD5 are too fast for password storage: a modern GPU can test billions of SHA-256 hashes per second.
