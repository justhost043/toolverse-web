URL encoding explained

URLs can only contain certain safe ASCII characters. Special characters like spaces, ampersands, equals signs, and non-ASCII letters must be percent-encoded (e.g. space becomes %20, ü becomes %C3%BC) before being placed in a URL. This ensures the URL is correctly interpreted by web servers, browsers, and APIs across all systems.

Component encoding vs full URI encoding

Component encoding (encodeURIComponent) encodes everything — including &, =, /, ?. Use this when encoding a single query parameter value, like a search term. Full URI encoding (encodeURI) preserves structural URL characters like ://, /, ?, and &. Use this when encoding an entire URL while keeping its structure intact.

Query string builder use cases

API developers use the query builder to construct URLs for REST endpoints without manually typing percent-encoded characters. Each key-value pair is automatically encoded and concatenated correctly. The result is a valid URL ready to use in Postman, curl, or a browser.
