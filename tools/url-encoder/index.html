What is URL encoding?

URLs can only contain certain safe ASCII characters. Special characters — spaces, ampersands, equals signs, non-English letters — must be converted to a percent-encoded format (like %20 for a space) before being used in a URL. URL encoding ensures the URL is valid and correctly interpreted by web servers and browsers.

Component vs full URI encoding

Use component encoding when encoding a single query parameter value — it encodes everything including characters like &, =, and /. Use full URI encoding when encoding an entire URL — it preserves structural characters like ://, /, ?, and & while still encoding unsafe characters.
