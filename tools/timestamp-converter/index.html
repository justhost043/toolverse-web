What is a Unix timestamp?

A Unix timestamp (also called epoch time or POSIX time) is the number of seconds that have elapsed since January 1, 1970 at 00:00:00 UTC, not counting leap seconds. It's a universal standard for storing and comparing times in software systems because it's a simple integer — no time zones, no ambiguity, no daylight saving complications. The same timestamp represents the exact same moment in time everywhere on Earth.

Seconds vs milliseconds

The original Unix timestamp counts seconds. Modern JavaScript and many APIs use milliseconds (Unix timestamp × 1000). A 10-digit number is seconds (current standard); a 13-digit number is milliseconds. This converter detects which format you've entered automatically.
