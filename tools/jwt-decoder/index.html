What is a JWT?

A JWT (JSON Web Token) is a compact, URL-safe format for transmitting claims between two parties, commonly used for authentication. It has three parts separated by dots: a header (specifying the algorithm), a payload (the claims/data), and a signature (verifying the token hasn't been tampered with). The header and payload are Base64URL-encoded JSON — readable by anyone, which is why JWTs should never contain sensitive secrets.

Common JWT claims

iss (issuer) identifies who created the token. sub (subject) identifies the user. exp (expiration) is a Unix timestamp after which the token is invalid. iat (issued at) records when the token was created. aud (audience) specifies who the token is intended for.
